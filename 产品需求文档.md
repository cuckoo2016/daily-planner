# 每日规划应用（Daily Planner App）产品需求文档（PRD）

**版本：** V2.0（Web前后端分离版）

---

# 一、文档基础信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 每日规划（Daily Planner） |
| 产品定位 | 轻量、简洁、高效的个人每日计划管理工具，帮助用户快速记录每日计划、设置提醒，养成规律规划习惯，提升时间利用率 |
| 目标用户 | 所有需要规划每日时间、记录待办事项、需要提醒功能的个人用户（学生、职场人、自由职业者等），核心需求为"简单记录+及时提醒"，不追求复杂功能 |
| 核心目标 | 1. 让用户快速、便捷地记录每日计划，无需复杂操作；2. 提供精准、灵活的提醒功能，确保用户不遗漏重要计划；3. 界面简洁、无冗余，降低用户使用成本 |
| 文档版本 | V2.0（Web版，支持多用户+管理员后台） |
| 适用平台 | Web 端（PC + 移动端浏览器响应式适配） |
| 技术栈 | 前端：Vue3 + Vite + TypeScript；后端：Node.js + Express + TypeScript；数据库：SQLite；认证：JWT |
| 用户角色 | 普通用户、管理员 |

# 五、产品核心需求（用户视角）

1. 能够快速进入每日计划页面，撰写当天的计划（支持多条计划，可分类、可编辑、可删除）；

2. 可以为每条计划设置提醒时间，到点收到推送提醒，不遗漏重要事项（Web版支持浏览器通知）；

3. 能够查看历史计划（过往每日的计划记录），支持搜索、筛选；

4. 计划可以标记完成/未完成，直观查看当日计划进度；

5. 界面简洁、操作流畅，无广告、无冗余功能，专注于"记录+提醒"核心需求；

6. **支持多用户**：每个用户独立账户，数据隔离，保护隐私。

# 六、整体产品架构

## 6.1 核心模块划分

- 首页（当日计划）：核心页面，展示、添加、编辑、删除当日计划，标记完成状态，设置提醒；
- 历史计划：查看过往每日计划，支持按日期筛选、关键词搜索；
- 提醒设置：统一管理所有计划的提醒，可开启/关闭提醒、修改提醒方式；
- 个人中心：基础设置（头像、昵称）、应用设置（主题切换）、帮助反馈。

## 6.2 页面跳转逻辑

- 启动应用 → 判断登录状态 → 已登录进入【首页】，未登录进入【登录/注册】页面；
- 首页 → 点击"历史" → 进入【历史计划】页面；
- 首页 → 点击"提醒设置" → 进入【提醒设置】页面；
- 首页 → 点击"我的" → 进入【个人中心】；
- 首页 → 点击"添加计划" → 弹出【添加计划】弹窗；
- 首页 → 点击单条计划 → 弹出【编辑计划】弹窗；
- 历史计划 → 点击某一天 → 进入【当日历史计划详情】页面。

# 七、详细功能需求（按模块划分）

## 模块1：首页（当日计划）（核心模块）

### 7.1.1 基础展示

- 顶部展示：当前日期（年月日+星期，如：2026年4月16日 星期三），字体突出，清晰可见；
- 计划列表：展示当日所有计划，按"提醒时间"升序排列（无提醒的计划排在末尾）；
- 单条计划展示：包含"计划内容"、"提醒时间（如有）"、"完成状态（未完成/已完成）"三个核心元素；
- 计划进度：顶部展示当日计划完成率（已完成数量/总数量），用进度条+百分比展示，直观清晰；
- 空状态：当日无计划时，展示引导文案（如："今天还没有计划，添加一条吧～"），并提供"添加计划"快捷按钮。

### 7.1.2 计划操作（添加、编辑、删除、标记完成）

#### （1）添加计划

- 触发方式：首页底部点击"+"按钮，弹出添加计划弹窗；
- 弹窗内容：
  - 输入框：提示文字"请输入今日计划..."，支持多行输入，最多输入200字；
  - 提醒设置：可选，点击"设置提醒"，弹出时间选择器（默认不设置提醒）；
  - 按钮："取消"（取消添加，不保存）、"保存"（保存计划，返回首页，计划自动添加到列表）；
  - 校验：输入框为空时，点击"保存"提示"请输入计划内容"，无法保存。

#### （2）编辑计划

- 触发方式：点击单条计划，弹出编辑计划弹窗；
- 弹窗内容：
  - 输入框：默认填充原计划内容，可修改，支持多行编辑；
  - 提醒设置：默认显示原提醒时间（如有），可修改、取消提醒；
  - 按钮："取消"（取消编辑，不修改）、"保存"（保存修改，返回首页，列表同步更新）。

#### （3）删除计划

- 触发方式：单条计划左滑，显示"删除"按钮；或编辑弹窗中添加"删除"按钮（二次确认）；
- 二次确认：点击"删除"后，弹出提示"确定要删除这条计划吗？删除后不可恢复"，提供"取消"和"确定"按钮；
- 删除后：计划从列表中移除，进度条同步更新。

#### （4）标记完成/未完成

- 触发方式：点击单条计划左侧的复选框（未完成→勾选→已完成；已完成→取消勾选→未完成）；
- 视觉反馈：已完成的计划内容显示"删除线"，颜色变浅，与未完成计划区分开；
- 进度同步：标记完成/未完成后，顶部进度条实时更新。

### 7.1.3 提醒设置（单条计划）

- 时间选择：点击"设置提醒"，弹出时间选择器（支持选择当日任意时间，精确到分钟）；
- 提醒状态：设置提醒后，计划列表中显示提醒时间（如：09:30），并添加"铃铛"图标；取消提醒后，图标和时间消失；
- 重复提醒：基础版暂不支持重复提醒（仅支持当日单次提醒），后续可迭代。

## 模块2：历史计划

### 7.2.1 基础展示

- 顶部筛选：日期筛选器（支持按"昨天、前天、近7天、自定义日期"筛选）；
- 历史列表：按日期倒序排列（最新的历史日期在前），每个日期展示"日期+当日计划完成率+计划数量"；
- 空状态：无历史计划时，展示文案"暂无历史计划记录"。

### 7.2.2 历史详情与操作

- 详情查看：点击某一日期，进入该日期的计划详情页，展示该日所有计划（已完成/未完成区分显示）；
- 搜索功能：顶部搜索框，支持按关键词搜索历史计划内容（模糊匹配）；
- 操作限制：历史计划仅支持"查看"，不支持编辑、删除（避免误操作，基础版暂不开放修改权限）。

## 模块3：提醒设置

### 7.3.1 全局提醒开关

- 总开关：顶部设置"全局提醒开启/关闭"，关闭后，所有计划的提醒均失效；开启后，按单条计划的提醒设置生效；
- 默认状态：用户首次登录后，全局提醒默认开启，需引导用户授予"浏览器通知权限"。

### 7.3.2 提醒管理

- 提醒列表：展示当日所有设置了提醒的计划，按提醒时间升序排列，包含"计划内容、提醒时间、开启/关闭开关"；
- 单个提醒开关：可单独开启/关闭某条计划的提醒，不影响全局开关和其他计划；
- 提醒方式：基础版支持"浏览器通知提醒"，后续可迭代"邮件提醒"。

### 7.3.3 提醒推送规则

- 推送时机：到达设置的提醒时间后，立即推送浏览器通知；
- 推送内容：通知标题"每日规划提醒"，通知内容"您有一条计划：[计划内容]，请及时处理～"；
- 推送跳转：点击通知，直接打开应用首页，定位到该条计划。

## 模块4：个人中心

### 7.4.1 基础信息

- 头像、昵称：支持上传头像、修改昵称（默认昵称"规划者"，默认头像为系统默认图标）；
- 使用统计：展示"累计使用天数"、"累计创建计划数"、"平均完成率"（增强用户粘性）。

### 7.4.2 应用设置

- 主题切换：支持"浅色主题/深色主题"，默认跟随系统主题；
- 清除缓存：清除应用缓存（如图片、历史记录缓存）；
- 关于我们：展示应用版本、开发者信息；
- 帮助反馈：提供反馈入口（如输入框，可提交使用问题、建议）。

### 7.4.3 账户管理

- 修改密码：用户可修改登录密码；
- 退出登录：安全退出当前账户。

# 十一、认证流程

## 8.1 注册流程

```
1. 用户点击"注册"
2. 输入用户名（唯一）、密码、确认密码
3. 后端验证：
   - 用户名不能为空
   - 密码长度 >= 6 位
   - 用户名不能已存在
4. 密码 bcrypt 加密后存储
5. 返回注册成功，跳转登录
```

## 8.2 登录流程

```
1. 用户输入用户名、密码
2. 后端验证用户名和密码
3. 验证成功：生成 JWT Token（24小时有效期），返回用户信息
4. 前端存储 Token 到 localStorage
5. 后续请求携带 Token（Authorization: Bearer <token>）
```

## 8.3 Token 验证

```
1. 每次请求携带 JWT Token
2. 后端中间件验证 Token 有效性
3. 无效/过期：返回 401，前端跳转登录
4. 有效：从 Token 提取 user_id 查询用户信息
5. 后续业务逻辑使用提取的 user_id，确保数据隔离
```
