# 每日规划（Daily Planner）Web 版开发计划

**版本：** V2.0（前后端分离 + 多角色系统）

---

## 一、项目概述

基于 PRD 文档，开发一款支持多用户的每日计划管理 Web 应用。

- **产品定位**：轻量、简洁、高效的个人每日计划管理工具
- **目标平台**：Web 端（PC + 移动端浏览器响应式）
- **用户角色**：普通用户（user）、管理员（admin）

---

## 二、技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Vue 3 + Vite + TypeScript |
| 状态管理 | Pinia |
| 路由 | Vue Router |
| HTTP 客户端 | Axios |
| UI 组件 | Element Plus |
| 后端框架 | Node.js + Express + TypeScript |
| 数据库 | SQLite (sql.js) |
| 认证 | JWT (jsonwebtoken) |
| 密码加密 | bcrypt-ts |
| 构建工具 | tsx (运行 TypeScript) |

## 五、API 接口设计

### 5.1 认证模块
| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/auth/register | 用户注册 | 否 |
| POST | /api/auth/login | 用户登录 | 否 |
| POST | /api/auth/logout | 用户登出 | 是 |
| GET | /api/auth/me | 获取当前用户信息 | 是 |

### 5.2 计划模块
| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/plans | 获取当日计划列表 | 是 |
| GET | /api/plans/history | 获取历史计划 | 是 |
| POST | /api/plans | 创建计划 | 是 |
| PUT | /api/plans/:id | 更新计划 | 是 |
| PATCH | /api/plans/:id/complete | 标记完成/未完成 | 是 |
| DELETE | /api/plans/:id | 删除计划 | 是 |
| GET | /api/plans/search | 搜索计划 | 是 |

### 5.3 用户模块
| 方法 | 路径 | 描述 | 认证 |

| GET | /api/user/profile | 获取用户资料 | 是 |
| PUT | /api/user/profile | 更新用户资料 | 是 |
| PUT | /api/user/theme | 更新主题设置 | 是 |
| GET | /api/user/stats | 获取使用统计 | 是 |

### 5.4 管理后台模块（需 admin 角色）
| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| GET | /api/admin/dashboard | 仪表盘数据 | admin |
| GET | /api/admin/users | 用户列表 | admin |
| GET | /api/admin/users/:id | 用户详情 | admin |
| PUT | /api/admin/users/:id | 更新用户 | admin |
| PATCH | /api/admin/users/:id/status | 启用/禁用用户 | admin |
| DELETE | /api/admin/users/:id | 删除用户 | admin |

## 十、注意事项

1. 第一个注册的用户自动成为 admin
2. 所有密码使用 bcrypt 加密存储
3. JWT Token 有效期 24 小时
4. 用户数据隔离，确保普通用户无法访问他人数据
5. 管理后台 API 需额外验证 admin 角色权限
6. 敏感操作记录到 operation_logs 表
